# 简历智能解析与岗位匹配推荐 (Resume-Job Matcher)

## 一、触发条件与适用场景

### 触发关键词
- **简历匹配** / **岗位匹配** / **匹配度分析** / **匹配评分**
- **简历解析** / **简历评估** / **简历诊断**
- **候选人评估** / **人岗匹配** / **岗位适配**
- **投递分析** / **面试推荐** / **筛选建议**
- **简历优化建议** / **JD对比** / **能力差距**

### 适用场景
| 场景 | 说明 |
|------|------|
| 求职者自评 | 求职者上传个人简历与目标岗位JD，获取匹配度评分和针对性优化建议 |
| HR初步筛选 | HR批量对比候选人简历与岗位要求，快速定位高匹配度候选人 |
| 猎头推荐 | 猎头评估候选人背景与客户岗位的适配程度，生成推荐报告 |
| 求职指导 | 职业顾问帮助学生/转行者分析自身与目标岗位的差距 |
| 简历优化 | 根据目标JD反向优化简历内容，提升面试通过率 |

### 不适用场景
- 简历纯格式排版
- 薪酬谈判与薪资分析
- 面试问题生成
- 批量自动化招聘流程编排

---

## 二、核心功能概述

1. **智能简历解析**：从非结构化文本中提取结构化信息
2. **JD要素提取**：解析岗位描述中的核心要求
3. **多维度匹配分析**：技能匹配、经验匹配、教育背景、项目经历、软实力五维评分
4. **差距诊断**：识别简历与JD之间的具体差距项，并标注严重程度
5. **优化建议生成**：针对每个差距项生成可操作的改进建议
6. **匹配报告输出**：生成结构化的Markdown匹配分析报告

## 三、操作步骤

### 步骤 1：获取并标准化输入
- 支持Markdown文本、纯文本、结构化JSON格式

### 步骤 2：简历智能解析
- 提取基本信息、教育经历、工作经验、技能、项目经历等

### 步骤 3：JD要素提取
- 提取岗位基本信息、必需技能、加分技能、经验要求、学历要求等

### 步骤 4：多维度匹配计算

**各维度权重配置**：

| 维度 | 默认权重 | 技术岗权重 | 管理岗权重 |
|------|----------|------------|----------|
| 技能匹配 | 30% | 40% | 15% |
| 经验匹配 | 25% | 25% | 30% |
| 教育背景 | 15% | 10% | 10% |
| 项目经历 | 20% | 20% | 20% |
| 软实力 | 10% | 5% | 25% |

### 步骤 5：差距分析与优化建议生成
- 高严重度：核心必要求缺失
- 中严重度：重要加分项缺失
- 低严重度：锦上添花项缺失

### 步骤 6：报告生成

## 五、匹配算法详解

### 5.1 技能匹配算法
```
技能匹配分 = (匹配技能数 * 1.0 + 部分匹配技能数 * 0.5) / (要求技能总数) * 100
```

### 5.2 经验匹配算法
```
经验匹配分 = min(100, (实际年限 / 要求最低年限) * 100) * 经验领域相关度系数
```

### 5.3 综合评分公式
```
总分 = Σ(维度i评分 × 维度i权重) + 亮点加分 - 风险扣分
```

## 六、已知限制

1. 当前版本主要支持中文简历和JD的分析
2. 解析准确度依赖于简历/JD的结构化程度
3. 技能关键词匹配无法完全替代语义理解
4. 经验工作的"质量"难以通过文本完全量化
5. 软实力评估依赖关键词匹配
